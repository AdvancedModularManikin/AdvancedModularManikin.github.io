\doxysubsection{Detailed Description}
\subsubsection*{template$<$class T, class L, class A, class U = void$>$\newline
class A\+M\+M\+::\+Amm\+Data\+Type$<$ T, L, A, U $>$}

\mbox{\hyperlink{namespace_a_m_m}{A\+MM}} template class for handling A\+M\+M-\/\+D\+DS topics.

\begin{DoxyAttention}{Attention}
This specialization of \mbox{\hyperlink{class_a_m_m_1_1_amm_data_type}{Amm\+Data\+Type}} should be used if a static or non-\/static method from a class instance is going to be used as the call back for subscribed events. An instance of \mbox{\hyperlink{class_a_m_m_1_1_amm_data_type}{Amm\+Data\+Type}} doesn\textquotesingle{}t have to be nested in that very object receiving subscribed events, however \mbox{\hyperlink{class_a_m_m_1_1_amm_data_type}{Amm\+Data\+Type}} needs to be aware of that object\textquotesingle{}s type to correctly translate function pointers.
\end{DoxyAttention}
The following template variables are examples of how to use this tempate. T = A\+M\+M\+::\+Assessment\+Pub\+Sub\+Type L = \mbox{\hyperlink{class_a_m_m_1_1_d_d_s___listeners_1_1_assessment_listener}{D\+D\+S\+\_\+\+Listeners\+::\+Assessment\+Listener}} A = A\+M\+M\+::\+Assessment U = Parent class (The class that contains a method intended to recieved subscribed event.)
